# 1.4-alpha
指完成了第四节要求的版本。先前的版本管理较为混乱。

## 本版已实现功能

- 基于服务端转发的 1 对 1 通信结构
- 图形化客户端主界面
- 在线用户列表显示
- 目标用户选择
- 文本消息发送
- 文件发送
- 离线语音录制与发送
- 实时语音通话
- 呼叫 / 接听 / 挂断 的基础通话状态机
- 联系人保存、备注、删除
- 服务端基础管理命令
- 项目结构已整理为 `src/` + `data/` 形式

## 当前目录说明

- `src/apps/`：客户端界面入口
- `src/server/`：服务端入口
- `src/core/`：协议、配置、音频、联系人等核心模块
- `data/contacts/`：联系人数据
- `data/received/`：客户端接收的文件/音频
- `data/recorded/`：本地录音
- `data/server_received/`：服务端收到的音频

## 运行方式

### 1. 启动服务端

直接双击根目录下：

- `start_server.bat`

或在终端运行：

```bash
python -m src.server.server
```

### 2. 启动客户端

直接双击根目录下：

- `start_client.bat`

或在终端运行：

```bash
python -m src.apps.client_stream_gui
```

## 基本操作指南

### 客户端

1. 启动客户端后，先输入用户名。
2. 连接成功后，在左侧列表中选择一个在线用户作为当前目标。
3. 可直接进行以下操作：
   - 发送文字
   - 发送文件
   - 录音 5 秒发送
   - 保存联系人 / 添加备注 / 删除联系人
4. 若要进行实时语音通话：
   - 先选择目标用户
   - 点击“呼叫”
   - 对方点击“接听”
   - 接通后再点击“开启实时通话”
   - 通话结束后点击“挂断”

### 服务端

服务端启动后，可在控制台使用以下命令：

- `help`：查看命令
- `list`：查看在线客户端
- `all <消息>`：向所有客户端广播文本
- `to <用户名> <消息>`：向指定用户发送文本
- `sendaudio <用户名> <路径>`：向指定用户发送音频文件
- `playlast`：播放最近收到的音频
- `playaudio <路径>`：播放指定音频文件
- `quit`：关闭服务器

**已修复问题**

- **同名登录踢人无通知**：之前 `server.py` 在新连接用了已存在的用户名时，直接关闭旧连接而不通知旧用户。现已在关闭旧连接前先向旧用户发送提示消息：“你的账号在其他地方登录，当前连接已断开”。

- **来电覆盖已有通话状态**：之前收到 `call` 消息后无论本地状态如何都会切换到 `RINGING`，导致 `CALLING` / `TALKING` 状态被覆盖。现已在 `client_stream_gui.py` 中加入忙线判断：若当前处于 `CALLING` 或 `TALKING`，自动向来电方回发 `hangup` 并提示“忙线已自动拒绝”，不改变当前通话状态。

- **挂断可发给任意选中用户**：之前 `hangup()` 在 `call_peer` 和 `ringing_from` 均为空时会回退到 `target_user`，即使没有建立通话也能给对方发挂断，造成对方状态错乱。现已移除该回退逻辑，只有真正存在通话对象时才允许挂断。

- **联系人目录不存在导致首次保存失败**：之前 `contact_manager.py` 的 `save()` 方法在写文件前未确保父目录存在，全新环境下首次保存联系人会抛出 `FileNotFoundError`。现已在 `save()` 中添加 `os.makedirs(..., exist_ok=True)`。

**已知局限**

文件和音频发送目前都是整文件读入内存，功能能跑，但扩展性偏弱。现在是一次性 `read()` 后直接发送；以后如果发送较大文件，会卡 UI、占内存，也不利于做进度提示。
